Github+HEXO部署个人博客教程（更新中）

前言

个人博客通常有两类选择。

一类是动态博客，如Wordpress、Typecho等，一类是静态博客，如Hexo等。

通俗来说，动态博客就是可以直接在网页端互动、发文，有服务器和数据库支撑，能够实时生成的网页。而静态博客则是生成一个完整的网页，无论何时何地点击，都会发送同样的页面。

综合考虑成本和需求后，我选择github+hexo部署静态博客。

很多hexo主题自带文章排版、tag分类等功能，数据统计及评论也可以依靠插件完成；后台更新文章虽然较为繁琐，但自由度也高，加上我的更新频率慢，所以影响不大。

作为第一次建站，低成本的尝试更加明智。如果能够长期坚持，需要拓展高级功能，我会再考虑升级。

在自学过程中，本人主要依靠CSDN的教程及ChatGPT辅助。过程中遇见的问题，也会在本合集之中一一讲述，希望能帮助有需要的人少走弯路。

基本思路

通俗而言，建设个站，首先是部署博客页面，其次要让它在网络上可访问——前者使用博客框架，后者则依靠服务器和域名。

Hexo是基于node.js生成静态博客的开源框架，支持Markdown和多数Octopress插件，拥有数百个主题模板方便下载和更换。这样就不用自己编写代码设计网页，也能够拥有美观的博客界面。

而GitHub是一个面向开源及私有软件项目的托管平台。用户可以通过Git等版本控制工具将自己的Hexo博客网页上传到GitHub上，然后通过GitHub提供的静态网站托管服务——也就是GitHub Pages直接部署到GitHub所提供的服务器上，以实现网站的发布和访问。

GitHub Pages的域名是以“username.github.io”的形式提供的，其中“username”是你的GitHub用户名。只要仓库中包含一个名为“index.html”的文件，就可以通过“username.github.io”访问你的网站。

像这样Github+HEXO部署的博客网页，既能免费获取GitHub提供的服务器资源，又能在GitHub上进行版本控制，非常方便。

一、环境准备

安装git

进入git官网，点击Downloads下载。

选择系统类型，例如Windows、64位系统


之后会跳出下载界面，下载完成后在文件夹中运行exe文件

安装设置在path设置中选择第二项，能同时在Git Bash和命令提示符cmd中使用Git。其他保持默认即可。

安装完成后，任意文件夹点击右键出现Git GUI Here和Git Bash Here即成功。

打开Git Bash Here

输入git，回车，得到下图

确认git安装完成！

安装node.js

下载及安装设置

进入node.js官网，下载lts（长期维护）版本

下载完成后在文件夹中运行安装包

安装设置有两项重点：
第一，nodejs默认安装目录可以自己选择，我保持默认。

第二，教程中有一种安装后手动修改环境变量的方式，我始终无法成功；卸载重装也容易因为卸载不干净出错。
但只要在安装设置中，勾选Add to PATH安装模式，用户环境变量和系统环境变量就会在安装的同时自动配置，无需额外修改，简单快捷。

除此之外保持默认即可。

如果想了解更多关于“手动配置环境变量”以及“卸载nodejs”的教程，可以参照以下链接：
Node.js下载安装、环境变量配置和卸载教程(Windows 10)
如何正确干净的卸载nodejs？

安装完成后，win+R在弹出的运行窗口内输入cmd，打开命令行。
在命令行窗口分别输入node -v，npm -v（注意空格），检查node.js和npm的版本号（Node自带npm）。
成功显示版本号则安装成功。如下：

C:\Users\Username>node -v
v18.15.0

C:\Users\Username>npm -v
9.6.2

路径设置

在命令行窗口分别输入
npm config get prefix
npm config get cache
检查当前全局安装路径和当前缓存路径：

C:\Users\Username>npm config get prefix
C:\Users\Username\AppData\Roaming\npm

C:\Users\Username>npm config get cache
C:\Users\Username\AppData\Local\npm-cache

在nodejs安装目录下，新建一个缓存文件夹（node_cache）和一个放置全局模块的文件夹（node_global），然后在命令行窗口分别输入
npm config set prefix "C:\Program Files\nodejs\node_global"
npm config set cache "C:\Program Files\nodejs\node_cache"
之后再次进行检察，显示为刚设置的路径则为成功：

C:\Users\Username>npm config set prefix "C:\Program Files\nodejs\node_global"

C:\Users\Username>npm config set cache "C:\Program Files\nodejs\node_cache"

C:\Users\Username>npm config get prefix
C:\Program Files\nodejs\node_global

C:\Users\Username>npm config get cache
C:\Program Files\nodejs\node_cache

至此，环境准备部分全部完成！
